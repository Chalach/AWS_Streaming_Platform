Motivation

Das Projekt wurde im Rahmen der Lehrveranstaltung Modern Networks erstellt. Die Architektur des Streaming Dienstes basiert auf AWS.

Die Streaming Plattform bietet dem Kunden ein einzigartiges Streaming Erlebnis. Dabei liefert die Webseite spannende und unterhaltsame Videos für groß und klein. Ohne Anmeldung kann der Kunde eine Liste von verfügbaren Videos aufrufen. Nach erfolgreichem Login kann der Kunde nun vollwertig auf das Streamingangebot zugreifen.

Aufgabenstellung

Bitte überlegen Sie selbständig ein kleines Projekt, das sie mit AWS realisieren möchten. Schwerpunkt/Thema sind frei. Sie können auch Softwareprojekte aus anderen Lehrveranstaltungen wiederverwenden und auf AWS migrieren.

Thema, Architektur, Lösungsweg sind frei - jedoch muss sich jedes Team in sinnvoller Art und Weise mit einigen ausgesuchten AWS Diensten intensiver auseinander setzen und damit eine Lösung bauen. Ziel ist es einen Prototypen zu bauen (kein fertiges Produkt). Teilfunktionalitäten dürfen durch Mockups/Dummies realisiert werden. Sie können Teile auf Basis von Open-Source Bausteinen oder Produkten realisieren, jedoch muss ein deutlich herzeig- und messbarer eigener Anteil enthalten sein.

Achten Sie darauf sich nicht zu übernehmen und planen Sie eine realistische Projektgröße. Bedenken Sie, dass Sie abhängig von Ihren Vorkentnnissen mit AWS bzw Cloud-Plattformen auch entsprechenden Einarbeitungs- und Lernaufwand haben. Überlegen Sie sich und Unterteilen Sie von Beginn an "must-have" und "nice-to-have" features, sollte die Umsetzung ins Stocken geraten.

Achten Sie darauf dass Sie die Lösung in maximal 10min im Rahmen einer Live-Online-Präsentation vorführen können. Angenommen Sie verwenden EC2 Auto-Scaling Groups dann soll die Funktionen des automatischen Skalierens mittels weiterer Ressourcen bei steigender Last dann auch Live demonstriert werden können. Planen Sie also von Beginn an auch eine entsprechende "Präsentierbarkeit" ein.

Neben der Live-Präsentation ist auch eine kurze Zusammenfassung des Projektes abzugeben (max 2 DIN-A4 Seiten). Diese muss zumindest eine kurze Lösungsbeschreibung und eine Architekturzeichnung enthalten, die die Komponenten, deren Interfaces und die Netzwerkarchitektur grafisch darstellt (subranges, gateways, firewalls, acls, etc).

Ihre Lösung muss folgende Anforderungen erfüllen:

  1. Verwendung von zumindest 2 der 3 verfügbaren AWS Interfaces (Management Console, CLI und/oder SDK/API)
  2. Verwendung von 2 oder mehreren AZ
  3. Verwendung von 3 oder mehr EC2 instances
  4. Verwendung von expliziten Security Groups auf jeder EC2 Instanz
  5. Verwendung eines oder mehrere Storage Services (EFS, EBS, S3, etc)
  6. Verwendung von EC2 Auto-Scaling Groups und/oder Elastic Load Balancer (ELB)
  7. Verwendung von subnets mit entsprechendem Routing Tabelle/ACLs

Sollten einzelne Anforderungen für sie nicht umsetzbar sein bzw. sollten diese bei Ihrem Projekt nicht sinnvoll einsetzbar sein, dann kann darauf - mit entsprechender Begründung im Abschlussdokument und bei der Präsentation - verzichtet werden.

Bitte beschreiben Sie Ihr Projekt in dem Forum auf Moodle unterhalb der Aufgabe (analog zum template). Ich habe die bisherigen Beschreibungen von Padlet bereits in dieses Forum übertragen.

Sobald ich in einem Kommentar die Beschreibung freigegeben habe, können Sie mit der Umsetzung beginnen.

Viel Erfolg!
